Testing HTML and PDF Creation with pod2html and pod2pdf

One-line summary:

Make an 8-bit color PNG with width 800 pixels and add this line in the POD:

    =for html  <img src="path_to_image/image.png">

Longer explanation:

Images for HTML and PDF generation should come from the single PNG image with 8 bit color and a width of 800 pixels. The PNG should be 8 bit-color as reported by running the file image.png command in unix. The height can be unconstrained.

Only the HTML code should be added in a POD document - I have a script that will automatically add the correct code for a PDF image afterwards. This is done because podchecker complains about the POD code for the PDF image.
